🕵️‍♂️ Why-Skill V2.0: Nexus Architecture
元提示词架构：人性逻辑动态编译器
“不要试图在烟雾弹里寻找真相，要去寻找缺失的物理约束变量。”

Why-Skill V2.0 不再是一个静态的提示词，而是一个基于 Nexus 元架构 的多智能体审计系统。它通过动态编译技术，针对每一起社交冲突实时“3D 打印”专属解剖刀。

🚀 核心进化：从“解剖刀”到“兵工厂”
在 V2.0 中，我们放弃了“万能提示词”的幻想，转而采用 Meta-Prompting (元提示词) 技术。

1. TCAS 四维物理审计协议
我们认为所有的社交诡辩都受限于物理世界，系统强制运行以下维度审计：

[T] Timeline (时间线)：锁定转场窗口，识别隐藏的第三方约会。

[C] Cost (成本)：追踪利益流向，识别谁在“隐形白嫖”集体资源。

[A] Authority (准入/许可)：探测背后的“影子决策者”（父母、配偶、老板）。

[S] Social Currency (社交币)：解构虚荣刚需，识别为了“出片/人设”而进行的逻辑绑架。

2. 调查先行原则 (Investigation First)
V2.0 强制执行“情报回填”逻辑：AI 不再盲目猜测。在输出结论前，系统会首先出具一份《核心情报缺口清单》，指导用户去物理世界获取关键证据。真相不在聊天记录里，真相在缺失的信息里。

🛠️ 运行流程 (Nexus Workflow)
输入案例：将混乱的聊天记录或冲突描述喂给 why-meta-engine.md。

动态编译 (Stage 1)：系统自动识别场景，生成【专属审计员 Prompt】并列出【缺失信息探针】。

情报回填：你根据探针去核实事实（或提供已知线索）。

终极审计 (Stage 2)：系统根据补全后的物理变量，出具冷酷的【终极审计报告】。

🔍 特色模块
元引擎 (The Nexus Engine)：自动调度芒格思维模型、博弈论、社会学插件。

主观降噪补丁：强制剥离用户情绪，将物理事实权重提升至 0.8。

逻辑双标探测：自动识别对方在博弈中使用的非对称准则（如：用“没规划”攻击你，但自己的方案也没规划）。
